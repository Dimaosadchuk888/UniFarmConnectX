Цель:
Создать полноценно выглядящий интерфейс UniFarm внутри Telegram Mini App, с визуальной структурой и всеми нужными окнами и блоками. Без подключения логики, API и блокчейна. Все поля, кнопки и формы — на месте, но БЕЗ обработчиков.

⸻

1. Общая структура 
 • Создать мини-приложение Telegram (WebApp) в папке /frontend
 • Стилизация: Фон:
 • Глубокий тёмно-серый/чёрный (#0F0F12 или аналогичный) — базовый фон интерфейса, создающий контраст.
 2. Фиолетовый (основной акцент):
 • Кнопка “Подключить кошелёк TON” и нижнее выделение в меню:
#A259FF или ближе к #B368F7
(яркий, чистый фиолетовый)
 3. Зелёный (для значений или акцентов):
 • В этом экране не виден, но ранее использовался для бонусов и фарминга:
Предположительно #4ADE80 или #00FF99 , плавные переходы, анимации
 • Использовать React или HTML+JS, поддержка адаптивности

⸻

2. Главные страницы и вложенные блоки

Dashboard (Главная) Добавить логотип надпись UniFarm в основном стиле приложения


 1. Приветствие
 • Текст: Привет, @username!
 • Подпись: “Сегодня отличный день для добычи токенов!”
 2. Текущий доход
 • Отображение: +0.0027 UNI / сек (заглушка)
 • Ежедневный итог: Заработано сегодня: 134 UNI (заглушка)
 3. График дохода
 • Мини-график (линейный, как в бирже)
 • Фиксированные точки (пример: каждые 2 часа)
 • Подпись: “UNI, накопленные за день”
 • Визуальный стиль: тёмный фон, фиолетовая линия, плавная анимация
 4. Boost статус
 • “Активные Boost: 0 / 4”
 • Подпись: “Ускорители увеличивают доходность”
 5. Check-in
 • Кнопка: “Получить ежедневный бонус” (заглушка, без логики)
 • Подпись: “Возвращайся каждый день, чтобы собирать бонусы!”

⸻

Технические ограничения:
 • Все элементы — только UI, без логики, API или базы
 • Кнопки не активны, данные не обрабатываются
 • График строится на заглушечных данных (фиксированный массив)
 • Интерфейс должен быть адаптивным и плавным


3. Farming

 • Заголовок: “Фарминг токенов UNI”
 • Поля:
 • Текущий доход: 0 UNI / сутки
 • Активный Boost (если есть): “—”
 • Ежесекундное начисление: 0 UNI / сек
 • Кнопка Boost → открывает блок прямо на этой странице:
      • 1 Основной пакет в UNI 
 • 4 пакета UNI + TON (1, 5, 15, 25 TON)
 • У каждого: название, описание, кнопка “Купить”
 • Заглушка под кнопку: “Оплата через TON будет доступна позже”
 • Внизу: “История фарминга” (пустая таблица) 


4. Missions
 • Заголовок: “Выполняй задания — получай UNI”
 • Список заданий (визуальные карточки):
 1. Подписаться на канал (заглушка кнопки)
 2. Пригласить 1 друга
 3. Подписка на YouTube
 4. Check-in дня
 • У каждого: иконка, название, кнопка “Выполнить” (заглушка), ожидаемая награда

Friends (Партнёрка)
 • Заголовок: “Партнёрская программа”
 • Поле с личной реферальной ссылкой (копировать)
 • Отображение: “0 друзей, 0 UNI доход”
 • Таблица (пустая): уровни, количество людей, доход

⸻

5. Важные требования
 • Всё должно быть видимо и кликабельно, но не выполнять реальных действий
 • Кнопки не должны отправлять данные — только UI
 • Не использовать TonConnect или Telegram SDK пока
 • Не добавлять фейковые useEffect, fetch и другие “пустые” вызовы
 • Поддержка мобильной адаптивности 

Wallet (Кошелёк)

 • Заголовок: “Ваш TON-кошелёк”
 • Поле: UQxxxxxxxxxxxxxxxxxxx (заглушка)
 • Кнопка: “Подключить кошелёк” (не работает пока)
 • Подраздел: Вывод средств
 • Поле “TON-адрес получателя”
 • Поле “Сумма для вывода”
 • Кнопка “Отправить заявку”
 • Подпись под кнопкой: “Заявка будет передана администратору”

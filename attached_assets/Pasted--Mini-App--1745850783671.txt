Техническое задание

Тема: Восстановление корректной маршрутизации страниц в Mini App после добавления Farming

Цель:
 • Обеспечить стабильную работу всех кнопок навигации.
 • Исправить переходы между страницами: Dashboard, Wallet, Friends, Missions, Farming.
 • Убедиться, что приложение правильно обновляет контент без ошибок.

⸻

Что нужно сделать:

1. Проверить функцию renderActivePage
 • Найти функцию или логику, которая отвечает за отрисовку активной страницы (renderActivePage или аналог).
 • Убедиться, что она корректно обрабатывает все значения state.activePage:
 • dashboard → Dashboard компонент
 • wallet → Wallet компонент
 • friends → Friends компонент
 • missions → Missions компонент
 • farming → Farming компонент

Если какие-то case отсутствуют или неправильно написаны — исправить.

⸻

2. Проверить обработчики навигации (кнопки меню)
 • Проверить логику onClick у кнопок навигации в Mini App:
 • Кнопка “Главная” → устанавливает state.activePage = 'dashboard'
 • Кнопка “Кошелек” → state.activePage = 'wallet'
 • Кнопка “Партнёрка” → state.activePage = 'friends'
 • Кнопка “Миссии” → state.activePage = 'missions'
 • Кнопка “Фарминг” → state.activePage = 'farming'

Если привязки нарушены — исправить.

⸻

3. Проверить работу роутера (если используется)
 • Если проект использует Wouter или React Router:
 • Убедиться, что для каждой страницы настроен правильный <Route path="...">
 • Пути типа /wallet, /friends, /missions, /farming работают корректно.

⸻

После выполнения:

Проверить вручную:
 • Переходы по всем кнопкам работают.
 • После клика страница мгновенно обновляется без ошибок.
 • В консоли нет ошибок навигации.
 • При переходах между Dashboard → Wallet → Friends → Missions → Farming интерфейс работает без перезагрузок.

⸻

Дополнительно:
 • Никаких новых маршрутов в серверной части настраивать не нужно (мы работаем внутри Mini App).
 • Структуру state и renderActivePage нужно держать максимально простой и читаемой.
 • Запрещено вносить глобальные изменения без необходимости.
